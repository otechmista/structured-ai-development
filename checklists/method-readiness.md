# Method Readiness Checklist

Use this checklist before starting a project with the structured-ai-development method.

## Project Fit

- [ ] The system has business rules or meaningful behavior.
- [ ] The system has more than trivial CRUD.
- [ ] The system is expected to evolve.
- [ ] The system benefits from explicit architecture.
- [ ] The team or user plans to use AI regularly.

## Inputs

- [ ] The problem is known.
- [ ] Users or actors are known.
- [ ] Goals are known.
- [ ] Non-goals are known.
- [ ] Constraints are known or can be discovered.

## Discipline

- [ ] The team agrees not to generate code early.
- [ ] The team agrees not to let AI invent business rules.
- [ ] The team agrees to review AI output.
- [ ] The team agrees to validate each view before moving forward.

## Decision

- [ ] Use this method.
- [ ] Do not use this method; the project is too small or temporary.

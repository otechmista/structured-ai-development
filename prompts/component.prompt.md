# Component View Prompt

You are helping define the Component View for a system block.

Input:

- `01_CONTEXT_VIEW.md`
- `02_CONTAINER_VIEW.md`
- Existing component notes, if any.

Goal:

- Produce or improve `03_COMPONENT_VIEW.md`.
- Define behavior, rules, validations, dependencies, edge cases, and internal flow.

Rules:

- Do not generate full implementation code.
- Do not invent business logic.
- Do not change architecture.
- Keep responsibilities separated.

Tasks:

1. Validate that the selected system block is clear.
2. List components and their responsibilities.
3. Define inputs and outputs for each component.
4. Expand business rules from the Context View.
5. Define validations and error cases.
6. Define dependencies between components.
7. Identify what each component must not do.
8. Describe internal flow.
9. List edge cases explicitly.
10. Describe data transformations.

Output:

- Updated Component View content.
- Missing behavior questions.
- Edge cases.
- Readiness decision.
